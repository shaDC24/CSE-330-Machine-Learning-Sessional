# Machine Learning Sessional Repository

This repository contains course materials, practice scripts, assignments, tutorials, and research references for the ML Sessional course.

## Repository structure

- `Online-1/`
  - Core course materials for the first online session.
  - Includes NumPy and Pandas tutorials, practice scripts, exam questions, dataset files, and assignment folders for A1/B1 batches.
  - `A1_19Batch/`, `B1_19batch/`: batch-specific CSV and Python files.
  - `A1+A2_question-21batch/` and `B1+B2_question-21batch/`: combined question sets with code and data.
  - `online-1_ML/`: additional online session scripts and question files.
  - Cheat sheets and sample lecture resources are included.

- `Online-2/`
  - Second online session materials.
  - Contains PyTorch tutorial code, sample scripts, and notes.
  - `Online_2_cheatSheet/`: cheat sheets and supplementary examples.
  - `Pytorch Tutorial/`: notebook-style PyTorch learning scripts and practice files.
  - `senior_resources/`: extra references.

- `Online-4/`
  - Advanced practice scripts and algorithm examples.
  - Includes supervised and unsupervised learning examples.
  - Files like `easy.py`, `medium.py`, `advanced.py`, and `Unsupervised_learning.py` are present.

- `Online3/`
  - Third online session resources and project notebooks.
  - Contains end-to-end examples for XGBoost, CatBoost, LightGBM, and other algorithms.
  - Includes `all_algorithms.py`, practice scripts, and dataset-focused notebooks.
  - `online3/`: nested folder with additional sample code.

- `Papers/`
  - Research papers and case studies grouped by topic.
  - Includes folders for Alzheimer, Kidney, PCOS, human activity detection, plant disease, and skin disease.
  - Also contains standalone research PDFs and documents.

## How to use

1. Open the folder in a Python-capable editor or notebook environment.
2. Install dependencies for the scripts you want to run.
   - Common libraries likely include: `numpy`, `pandas`, `matplotlib`, `scikit-learn`, `seaborn`, `torch`, `xgboost`, `lightgbm`.
3. Run Python scripts from the appropriate session folder.
4. Open `.ipynb` notebook files in `Online3/` with Jupyter or VS Code Notebook.

## Notes

- The folder organization reflects course sessions and assignment sets.
- `Papers/` is a reference collection and does not contain runnable code.
